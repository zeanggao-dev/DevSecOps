## 中文版

本仓库构建了一个面向企业级实践的 DevSecOps 内容体系，涵盖安全设计、自动化交付、运行时保障与治理评估等关键环节。它不仅是一个演示项目的集合，更是一份可以兑现的工程能力蓝图，旨在帮助组织和团队快速搭建可信赖的安全交付平台。

本仓库强调三大核心价值：

1. 安全贯穿全流程：将安全策略嵌入开发、测试、构建、部署和运维环节，形成从代码到运行时的完整闭环。
2. 自动化驱动交付：通过流水线、容器与基础设施即代码，减少人工重复操作，提升交付稳定性与可审计性。
3. 开源导向协作：以开放方式组织内容，鼓励标准化文档、贡献流程与社区反馈，增强项目可持续性。

📘 本仓库适合用于：

- 搭建标准化 DevSecOps 体系框架
- 验证企业级安全交付流水线
- 构建云原生与容器化运维能力
- 组织安全自动化培训与实战演练

🔍 关注点包括：

- 安全策略与安全扫描集成
- CI/CD 与流水线自动化
- Kubernetes 与容器平台部署
- 运行时监控与漏洞治理
- 开源治理与协作流程

✍️ 本仓库旨在提供一种可持续的学习与实践路径，便于技术团队在真实业务环境中复用与扩展，形成具有行业参考价值的 DevSecOps 规范。

## English Version

This repository is designed as a professional DevSecOps resource hub for organizations and practitioners who want to build secure, automated, and cloud-native delivery capabilities. It goes beyond code samples to provide a practical blueprint for implementing secure delivery, runtime protection, and collaborative open source governance.

The repository is built around three core principles:

1. Security across the lifecycle: embed security controls into development, testing, build, deployment, and operations to create a complete and traceable delivery chain.
2. Automation-first delivery: use pipelines, containers, and infrastructure-as-code to reduce manual work, improve consistency, and ensure auditability.
3. Open source collaboration: organize content transparently, document contribution processes, and encourage community-driven improvements.

📘 This repository is suitable for:

- Establishing a standardized DevSecOps framework
- Validating enterprise-grade secure delivery pipelines
- Building cloud-native and containerized operational capability
- Supporting security automation training and hands-on exercises

🔍 Focus areas include:

- Security policy and scan integration
- CI/CD and pipeline automation
- Kubernetes and container platform deployment
- Runtime monitoring and vulnerability management
- Open source governance and collaboration workflows

✍️ The intent is to provide a sustainable learning and implementation path that teams can adapt to real-world environments, creating a reference-quality DevSecOps standard.

## 设计理念 / Design Principles

- 以“安全、自动、可复用”为原则，兼顾研发效率与运行安全。
- 以“示例+说明+流程”组合方式，使内容兼具学习价值和落地能力。
- 以“开源协作”为目标，支持多方参与、持续改进与经验沉淀。

- Security, automation, and reusability guide the approach.
- Example-driven design ensures both educational value and production applicability.
- Open collaboration enables broad participation, continuous improvement, and knowledge retention.
